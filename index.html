What is React?

  • The V in MVC
  • Developed by Facebook and open-sourced

Why use React?

  • Fast
  • Declarative
  • Easy to understand and maintain at scale
  • Flexibility - can be used with your favorite logic/routing libraries. (But React router is great)
  • In fact, it's so flexible that it can be used on the server side as well as the client side (for isomorphic apps)!
  • And of course it can be used for cross-platform mobile development with React Native

Should I use React?

  • Great for dynamic sites with lots of data
  • Also great if you want to use vanilla JS and ES6, rather than learn a new DSL (domain specific language)
  • May be overkill for very small, static sites, since it requires more explicit code than other frameworks
  • Some people prefer to keep JS and HTML separate (and prefer Vue or AngularJS) - but don't knock JSX until you've tried it!

What is a Component?

  • Components are the building blocks of React
  • They are classes that extend Component
  • They always implement a render function, which outputs HTML

		class Greeting extends Component {
		  render() {}
		}

What is a Class?

  • A class is a new Javascript feature, added in ES6 (ECMA2015).
  • It is literally a function with syntactic sugar, that behaves similarly to classes in object-oriented languages (like Java).
  • A class can extend another class with the "extends" keyword. If it does so, it's constructor needs to use the word super().
  • We'll review all of this later!

		class Greeting extends Component {
			// we won't be modifying the constructor much so we won't be seeing this throughout the course.
			constructor() {
				super();
			}
		}

MDN docs

What is JSX?

  • Think of it as code that looks like HTML, and is transpiled into Javascript that generates HTML.
  • (If you're curious about the X, the name is short for Javascript XML.)
  • One (big) benefit is that you can catch more template errors at compile time, rather than on page load
  • Making smaller, reusable components is also easier, versus other frameworks where a new template file and JS file need to be created for each new component.
  • Read more about the benefits here

What is JSX?

  • Almost fully identical in appearance to HTML - except:
  • className - Instead of writing <li class="red">, you would write <li className="red">. This is because class is now a reserved word in Javascript.
  • key - To optimize list display performance, where lists can be reordered/changed, React asks for a key attribute that points to a unique ID for each list element.
  • Synthetic events - Some event handlers (onSubmit, onClick) have different names or implementations in different browsers. The HTML created by your code will be browser specific (onSubmit will turn into submit, etc), simplifying cross browser compatibility.
  • And some other exceptions we w get to later!

What is JSX?

Syntax - Hello World example


		class Greeting extends Component {
			render() {
				return (<div>Hello World!</div>);
			}
		}

Now, in another component, we can use the Greeting component, with the self-closing tag:


		class Layout extends Component {
			render() {
				return (<Greeting />);
			}
		}

What is JSX?

Syntax - Using JS in JSX

Javascript can be used in JSX as long as it is surrounded by single curly braces ({}).


		class Greeting extends Component {

			render() {
				const message = 'Hello World!';
				return (<div>{message}</div>);
			}
		}

What are transpiling and compiling?

Transpilation is the process of turning Javascript that cannot be understood by the browser (ES6, JSX) into the Javascript understood by browsers today (ES5).

It is usually done with BabelJS.

Compilation is the process of turning one language into another, transpiling is compiling between two languages that are similar. So you can use compilation and transpiling interchangably when talking about turning ES6/JSX into Javascript.

What is the Virtual DOM?

First things first: the browser DOM is an in-memory abstraction of HTML

DOM stands for Document Object Model

Updating the DOM requires browser manipulation, which is slow

The virtual DOM is an in-memory representation of the DOM. Updating the virtual DOM does not change the browser, so it is much faster.

What is the Virtual DOM?

We use ReactDOM.render to add a component to the VirtualDOM.

Whenever the page loads, or data changes after a page load, React creates a new VirtualDOM, and compares it to the old one. It takes the changes between them and applies them to the real DOM. This results in less browser manipulation, and faster performance. 


		class Greeting extends Component {
			render() {
				return (<div>Hello World!</div>);
			}
		}
		ReactDOM.render(<Greeting />, document.getElementById('Greeting'));

The first argument to ReactDOM.render() is the JSX to render, and the second is an element on the page to insert it in (selected with vanilla JS).

What are Props?

  • Props (properties) are attributes passed down from one component to another
  • A component's props can be accessed through this.props in the render function
  • A component's props are read-only. HOWEVER! Functions can be passed down as props as well. We'll see why this is important soon.

Hello World with Props


		class Greeting extends Component {
			render() {
				return (<div>{this.props.message}</div>);
			}
		}
		class App extends Component {
			render() {
				const message = 'Hello World!';
				return (<Greeting message={message} />);
			}
		}
		ReactDOM.render(<App />, document.getElementById('app'));

What is state?

  • State is used to store information about the component that can change over time. More?
  • The idea behind React is that your code has everything you need to know to display your views, given a specific state.
  • State is set at the parent component, and then passed down to children as props.
    • 
		class Greeting extends Component {
			render() {
				return (<div>{this.props.message}</div>);
			}
		}
		class App extends Component {
			state = {
				message: 'Hello World!'
			};
			render() {
				return (<Greeting message={this.state.message} />);
			}
		}
		ReactDOM.render(<App />, document.getElementById('app'));

    What is setState?

    To change a component's state, use this.setState(newState).

    But how can we do this from child components? Pass FUNCTIONS as props!

    
		class Greeting extends Component {
			render() {
				return (<div onClick={this.props.changeMessage.bind(this)}>{this.props.message}</div>);
			}
		}
		class App extends Component {
			state = {
				message: 'Hello World!'
			};
			render() {
				return (<Greeting message={this.state.message} changeMessage={this.sayGoodbye} />);
			}
			sayGoodbye(event) {
				event.preventDefault();
				this.setState(Object.assign({}, this.state, { message: 'Goodbye' }));
				// Equivalent to this.setState({ message: 'Goodbye' })
			}
		}
		ReactDOM.render(<App />, document.getElementById('app'));

    mdn

    Code Breakdown!

    Let's go over the last example piece by piece:

    • We added a sayGoodbye method to the App class, and used it as an event handler in the Greeting class. Event handlers are functions called from onClick, onSubmit, etc. 
      
		sayGoodbye(event) {
			event.preventDefault();
			this.setState({message: 'Goodbye'});
		}
    • Event handlers do not need to be invoked (have parentheses at the end). But when a click happens, the event handler is called with an event as the argument. To prevent side effects (like form submissions from pressing a button in a form), use event.preventDefault() (above). 
      
		render() {
			return (<div onClick={this.props.changeMessage.bind(this)}>{this.props.message}</div>);
		}

    Object.assign

    Object.assign is new to Javascript with ES6. It takes the first object (usually empty) and then updates it with attributes from any following objects, in order. This is a great way to create new objects based on existing ones concisely.

    
		this.setState(Object.assign({}, this.state, { message: 'Goodbye' }));
		// Would be especially useful with a large state, with many attributes

    Creating lists with JS's map

    Reminder: We need to set the key attribute in each node in a list of nodes, for optimum React performance.

    
		class App extends Component {
			state = {
				messages: [
					{id: 'a', value: 'Good morning'},
					{id: 'b', value: 'Good evening'},
					{id: 'c', value: 'Good night'},
				]
			}
			render() {
				return (
					<div>
					<span>Greetings!<span>
					<{this.state.messages.map(message =>
						(<span key={message.id}>message.value</span>)
					)}>
					<div>
				);
			}
		}

    For those newer to ES6: What are arrow functions?

    Arrow functions are a new, improved syntax for functions

    
		// Simplified (for when there's on argument and only one call in the function body)
		this.state.messages.map(message =>
						(<span key={message.id}>message.value</span>)
		);
		// More common
		this.state.messages.map((message, index) => {
			console.log(index);
			return (<span key={message.id}>message.value</span>);
		});
		// ES5 equivalent
		this.state.messages.map(function(message) {
			return (<span key={message.id}>message.value</span>)
		});

    One benefit of using arrow functions is that they do not define their own `this`, so you don't need to use `var that=this` with them anymore. And they are more concise than ES5 functions. Yay!

    Read more about them at the MDN docs.

    Pros and cons of React

    • Fast
    • Declarative - Easy to understand and maintain at scale (why?)
    • Flexible
    • Cross-platform mobile development (React Native)

    Classes + Components

    • Classes: ES6 addition, allows object oriented programming (with JS functions behind the scenes)
    • Components: building blocks of React
    • Your components extend Component

    JSX

    • Javascript that looks like HTML, and gets compiled to javascript that renders HTML
    • See what compiled JSX looks like!

    Props

    • Attributes passed down from one component to another
    • A component's props can be accessed through this.props in the render function

    State

    • Data stored in parent component, passed down to child components through props
    • Updated with this.setState (this.state and this.props are immutable)

    Why is immutability important?

    • Makes it easier to keep track of changes, test, and use time travel!
    • Thus we use this.setState instead of this.state = ...
    • Helpful methods for copying data: Array.slice, Array.concat, and Object.assign

    Lifecycle hooks

    • Docs
    • componentWillMount**: Function called before component is rendered. This is the most commonly used lifecycle hook (AJAX calls, timers)
    • componentDidMount: Function called after component is rendered. Useful for third party integrations if really needed (jQuery plugins)
    • componentWillUnmount: Function called before component is removed from the DOM
    • componentDidUnmount*: Function called after removal from DOM. Useful for cleanup (removing timers, etc.)
    • Official codepen
    • My codepen

    Functional Components

    For child components where you only need the render function, and do not need to modify the lifecycle methods, you can write your component more concisely! (Docs)

    
		function Welcome(props) {
		  return <h1>Hello, {props.name}</h1>;
		}

		const element = <Welcome name="Sara" />;
		ReactDOM.render(
		  element,
		  document.getElementById('root')
		);

    Official Codepen

    Composition vs Inheritance

    • Docs
    • Components are very flexible - you can even pass in their contents with props.children!
    • You can also pass JSX elements as your own props
    • And you can extend your own components!
    • Composition is always preferred over inheritance in React - make components as reusable as possible!

    Forms

    • Docs
    • Controlled Components (recommended!)
    • Uncontrolled Components

    Controlled Components

    • Forms have internal state, but controlled components keep React state in sync with form state
    • In practice: value for all types of inputs (including initial/default values), and run setState onChange
    • Examples: 
      
		<input type="text" value={this.state.value} onChange={this.handleChange} />
		<textarea value={this.state.value} onChange={this.handleChange} />
		<select value={this.state.value} onChange={this.handleChange}>
			  <option value="grapefruit">Grapefruit</option>...
		</select>
    • Official codepen 1 and 2
    • My codepen 1 and 2
    • (Form Docs)

    Uncontrolled Components

    • Docs
    • Use refs to store internal form state, and use setState on submit
    • Use defaultValue to set initial values in form elements (input, select, etc.)
    • Usually not recommended if you can use controlled components (or Redux)
    • "Controlled and uncontrolled form inputs in React don't have to be complicated"
